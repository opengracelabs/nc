# v0.2.0 ‚Äî Knowledge Facts, Relationships, and API

**Tag:** `v0.2.0-milestone-002`
**Date:** 2026-05-31

## Purpose

Introduced the knowledge layer: sourced facts, typed relationships, a controlled concept vocabulary, and a governance API for editorial review. Every claim is traceable to its source field and replayable from raw place data.

## Capabilities

- **Knowledge worker** ‚Äî pure field-mapping extraction with PROV-O provenance on every fact (`source_field` traces to exact place column); confidence scoring by source authority (UNESCO=0.95, Wikidata=0.80); claim/release poll loop with supersede-on-change and reactivate-on-replay
- **`co_inscribed_with`** relationships derived via join query during extraction
- `GET/POST /knowledge/concepts`, `GET /knowledge/concepts/{id}`
- `GET /knowledge/facts`, `GET /knowledge/facts/{id}`, `PATCH /knowledge/facts/{id}` ‚Äî dispute/retract governance (`active‚Üídisputed‚Üíretracted`)
- `GET /knowledge/relationships`
- `GET /knowledge/places/{place_id}/knowledge` ‚Äî aggregate facts + relationships

## Tests

- 130 tests passing (51 new replay + 30 new knowledge API + 49 carried from v0.1.0)
- Every fact tested for `provenance.source_field` traceability
- All 50 sites produce deterministic, replayable facts
- Concept URI links verified for all OUV criteria and heritage types
- Governance state machine covered: dispute and retract transitions

No performance-specific work in this release. Index strategy deferred to v0.2.1 following scale review.

## Schema Changes

Migrations 08‚Äì12:

- **`concepts` + `concept_aliases`** ‚Äî controlled vocabulary with anti-cycle trigger
- **`facts`** ‚Äî atomic sourced claims with closed predicate vocabulary `CHECK` constraint, content-addressable `UNIQUE` index (place, predicate, language, value::text), stale-facts trigger on authority field change
- **`relationships`** ‚Äî typed edges (place‚Üíconcept, place‚Üíplace) with self-reference guard
- **`places`** ‚Äî `knowledge_extracting`, `last_knowledge_extracted_at`, `knowledge_score` added
- **Seed** ‚Äî OUV criteria i‚Äìx, heritage types, institutional actors

## Known Limitations

- `draft` fact status exists in schema but is unused ‚Äî worker inserts directly as `active`
- GIN provenance index projected at 259 MB at 50K sites ‚Äî needs removal
- Multi-value predicates (e.g. `country_code`) risk incorrect supersession on replay
- Relationship governance endpoints (`GET/PATCH /relationships/{id}`) not yet implemented
- `knowledge_score` not written atomically with extraction release
