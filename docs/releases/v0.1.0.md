# v0.1.0 — UNESCO WHC Pipeline

**Tag:** `v0.1.0-milestone-001`
**Date:** 2026-05-31

## Purpose

First end-to-end pipeline release. Ingests, preserves, and serves UNESCO World Heritage site records through a five-stage pipeline from raw discovery to searchable API.

## Capabilities

- **Ingestion worker** — validation, evidence preservation, atomic promotion with two-transaction claim pattern (`FOR UPDATE SKIP LOCKED`), stale-asset recovery
- **Preservation worker** — checksum/size verification, quarantine on mismatch, `preserving` asset status
- **Discovery store** — 7 additional WHC fields: `statement_of_ouv`, `justification`, `transboundary`, core/buffer area, `spatial_precision`, `unesco_ref_id`
- `PATCH /places/{id}` — lifecycle governance endpoint
- `GET /places/search` — ILIKE + filter composition
- `GET /places/{id}/assets`

## Tests

- 58 tests passing

- 21 replay tests covering individual subsystems

## Performance

No performance-specific work in this release. Baseline established.

## Schema Changes

- `06_search.sql` — `tsvector` + GIN index, `update_place_search_vector` trigger
- `07_milestone_001.sql` — `preserving` status added to assets `CHECK` constraint

## Known Limitations

- Knowledge extraction not yet implemented; `knowledge_score` and related fields unpopulated
- Search uses `ILIKE` — no trigram or full-text ranking
- No relationship or concept vocabulary
