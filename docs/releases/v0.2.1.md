# v0.2.1 — Knowledge Hardening

**Tag:** `v0.2.1-knowledge-hardening`
**Date:** 2026-06-01

## Purpose

Hardening release resolving all blockers and high-priority issues identified in the MILESTONE-2.5 scale review. Extends governance to relationships, ships vocabulary v1, corrects replay correctness bugs, and validates the pipeline at 10,000 sites.

## Capabilities

- **Relationship governance** — `GET /knowledge/relationships/{id}` and `PATCH /knowledge/relationships/{id}`; full `_REL_TRANSITIONS` state machine (`activate`, `dispute`, `retract`); `agent_notes` for governance audit trail; `get_place_knowledge` returns both subject-side and object-side relationships
- **Fact governance** — retraction of disputed facts requires a `reason` string, enforced at API layer
- **Vocabulary v1** — canonical seed (`09_vocabulary_v1_seed.sql`) with architecture documentation
- **Replayability** — `build_place_concept_relationships` is deterministic; `upsert_facts` uses `predicate <> ALL` guard so multi-value predicates don't supersede each other; `upsert_relationships` uses set-based replay insert
- **Stale-knowledge triggers** — `flag_stale_facts_on_place_update` resets `last_knowledge_extracted_at` to `NULL` on authority field change, forcing immediate re-queue; `retract_stale_co_inscribed` retracts `co_inscribed_with` edges when `inscription_year` moves `active→superseded`
- **`knowledge_score`** written atomically with extraction release inside `release_places`

## Tests

- 185 tests passing
- 10,000 site load path validated (`scripts/load_test_knowledge_worker.py`)
- New replay coverage: multi-value predicate upsert, relationship idempotency, retraction with reason, migration index assertions

## Performance

- Dropped oversized/unused indexes: GIN provenance index (259 MB at 50K sites), `idx_facts_place_pred`, `idx_rel_subject/predicate/status`, `idx_facts_status`, `idx_places_knowledge_queue`
- `fillfactor=70` on facts and relationships to mitigate UUID v4 heap fragmentation
- Added covering indexes: `idx_facts_active_predicate_created_at`, `idx_relationships_active_place_co_inscribed`, `idx_places_description_trgm`, `idx_places_active_updated_at_desc`
- `build_co_inscribed_relationships` bounded to current batch `place_ids`; `LEAST`/`GREATEST` canonical ordering for correct `ON CONFLICT` deduplication
- `process_places` wrapped in `async with conn.transaction()` — prevents partial writes on mid-batch connection failure

## Schema Changes

Migration 14 (idempotent upgrade path):

- `draft` status removed from facts `CHECK` constraint — `active` is the only insert state
- `concept_aliases.updated_at` column and `trg_concept_aliases_updated_at` trigger added
- `fillfactor` storage parameters set on facts and relationships
- Oversized indexes dropped; performance indexes added (see Performance above)

## Known Limitations

- Wikidata ingestion not yet implemented; confidence scoring for non-UNESCO sources is defined but untested against live data
- Relationship governance covers place→concept and place→place edges; asset-linked relationships not yet modelled
- Vocabulary v1 covers OUV criteria and heritage types; geographic and administrative concept classes deferred
