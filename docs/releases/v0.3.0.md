# v0.3.0 — Taxon Discovery

Milestone 3 adds public-domain illustration opportunity discovery without changing the frozen architecture. The pipeline starts with a place, uses GBIF and Wikidata taxa as search handles, ranks opportunities by public-domain illustration value, and emits BHL search targets. This stage does not ingest BHL assets and does not infer rights.

## Added

- Source roles: BHL is primary discovery, GBIF is validation, Wikidata is context.
- Golden Age priority for 1750-1900 publications and priority illustrator scoring.
- Illustration opportunity schema with explicit rights evidence, place links, and concept ownership.
- Taxon discovery schema: runs, ranked candidates, GBIF/Wikidata evidence, and BHL search targets.
- Taxon discovery API for ranked taxa and place-level BHL search targets.
- Deterministic illustration opportunity scorer that uses taxa as search handles and caps occurrence-count influence.
- BHL illustration asset contract: Illustration Asset -> Species -> Place.
- Source registry support for Biodiversity Heritage Library (`bhl`).
- Asset type support for `bhl_illustration`.
- Research schema: `research_outputs`, `research_statements`, and `research_statement_evidence`.
- Evidence contract: every research statement must have source evidence, a fact, and a relationship through non-null foreign keys.
- Research worker: deterministic template composition from active facts and active relationships only; unsupported statements are skipped.
- Research API: list/detail endpoints plus governance actions for approve, publish, reject, dispute, and retract.
- Replay tests for schema enforcement, worker determinism, unsupported-statement rejection, API evidence detail, governance transitions, and Compose wiring.

## Constraints

- Not a biodiversity inventory system.
- Place-centered public-domain illustration discovery and commerce platform.
- Output is Illustration Opportunity, not Species.
- Do not optimize for species frequency, popularity, or raw occurrence counts.
- No BHL asset enters commerce unless rights are explicitly verified as Public Domain or CC0.
- Not a book ingestion program.
- Not a full literature graph.
- Optimize for commercial public-domain asset reuse.
- No Kubernetes, Neo4j, vector database, new datastore, or new queue.
- No unsupported research statements.
- Human review is required before publication.
