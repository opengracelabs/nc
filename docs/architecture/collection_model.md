# Collection Model

Collection #000001 turns verified BHL illustration assets into governed commercial sets.

Collections are not discovery candidates. They are publication-ready groupings of assets that have already passed rights verification and BHL asset ingestion.

## Rules

- Collections must have a stable slug, title, type, lifecycle status, provenance, and governance notes.
- A collection can be approved or published only when it has at least one place connection, at least one verified asset, and a reviewer.
- Collection assets must be BHL illustration assets with explicit Public Domain or CC0 rights.
- If a collection asset references an illustration opportunity, the asset must be linked to that same opportunity.
- Exports must record path, checksum, byte size, creator, format, status, and provenance.

## Lifecycle

Draft → Approved → Published

Draft → Rejected

Approved or Published → Disputed → Retracted

## Export

The initial export format is `collection_manifest_v1`: a deterministic JSON manifest ordered by collection asset sequence. Commerce ZIP export is reserved in the schema but not generated by Collection #000001.
